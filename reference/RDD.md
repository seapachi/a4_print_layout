# 要件定義書（RDD）

## 1. 文書情報
- 文書名: 画像印刷レイアウトWebアプリ 要件定義書
- 作成日: 2026-03-03
- 対象: iPhone 17 + Chrome で利用するWebアプリ
- 版数: v0.2（ワイヤー案・実装仕様追記）

## 2. 背景・目的
ネット上で取得した様々なサイズの画像を、iPhone上で手軽に印刷向けレイアウトへ整形し、A4印刷可能なPDFとして出力できるようにする。

## 3. スコープ
### 3.1 対象
- 端末内の複数画像を選択する
- 選択した画像をA4ページ上に配置する
- レイアウト結果をPDFとして生成・保存（または共有）する

### 3.2 対象外（現時点）
- クラウド保存連携（Google Drive / iCloud Drive API連携等）
- ユーザーアカウント機能
- 画像編集（色補正、背景除去など）
- 複数ページPDFの高度編集（ページ結合・並べ替えUI）

## 4. 想定ユーザー
- 個人ユーザー（家庭用プリンタ利用）
- ネット画像を印刷したいユーザー

## 5. 利用環境
- デバイス: iPhone 17
- OS: iOS 最新版（リリース時点）
- ブラウザ: Chrome 最新版（iOS版）
- 通信: オフラインでも基本利用可能（画像選択〜PDF生成まで）

## 6. 用語定義
- A4: 210mm x 297mm
- 2分割: A4内に同サイズ領域を2つ配置するレイアウト
- 4分割: A4内に同サイズ領域を4つ配置するレイアウト
- 6分割: A4内に同サイズ領域を6つ配置するレイアウト
- 8分割: A4内に同サイズ領域を8つ配置するレイアウト
- 印刷品質: 文字や輪郭が実用上十分に鮮明な状態（低解像度化を最小化）

## 7. 機能要件
### FR-01 画像選択
- 端末内の画像を複数枚選択できること
- 対応形式は最低限 `JPEG` / `PNG` とすること

### FR-02 レイアウトテンプレート選択
- 以下の印刷サイズ（レイアウト）を選択できること
  - A4全面（1枚配置）
  - A4 2分割
  - A4 4分割
  - A4 6分割
  - A4 8分割

### FR-03 画像配置
- 選択したテンプレートに応じて画像を配置できること
- 画像の縦横比を維持して配置すること（歪みなし）
- 余白・トリミング発生時はプレビューで視認できること

### FR-04 プレビュー
- PDF生成前にA4上の配置結果をプレビューできること
- プレビューはiPhone縦画面で確認しやすいUIであること

### FR-05 PDF生成
- A4サイズのPDFを生成できること
- 画像は可能な限り高解像度を維持した状態で埋め込むこと
- 生成後に「保存」または「共有」操作へ遷移できること

### FR-06 エラーハンドリング
- 画像読み込み失敗時にユーザーへ明確なエラーを表示すること
- メモリ不足などでPDF生成に失敗した場合、再試行可能な導線を提供すること

## 8. 非機能要件
### NFR-01 品質（印刷品質）
- 出力PDFはA4印刷時に実用上の鮮明さを保つこと
- 不要な再圧縮や過度なリサイズを避ける実装とすること

### NFR-02 性能
- 10枚程度の画像選択時、一般的な利用環境で実用的な待ち時間でプレビュー/生成できること

### NFR-03 ユーザビリティ
- 主要操作（画像選択 → レイアウト選択 → PDF生成）を3ステップ程度で完了できること
- スマートフォン単手操作を考慮したUI配置とすること

### NFR-04 互換性
- iPhone 17 の iOS版Chromeで正常動作すること
- 主要機能はSafari（iOS）でも動作する設計が望ましい

### NFR-05 セキュリティ・プライバシー
- 選択画像は原則として端末内処理し、外部送信しないこと（将来機能を除く）

## 9. 画面・操作フロー（案）
1. トップ画面で「画像を選択」
2. 選択後に「レイアウト（A4 / 2分割 / 4分割 / 6分割 / 8分割）」を選ぶ
3. プレビューで配置確認
4. 「PDFを生成」
5. 保存または共有

## 10. 制約・前提
- iOS版ChromeはWebKit制約を受けるため、ブラウザAPI利用可否を事前検証する
- 端末メモリ制約により、超高解像度画像多数投入時は分割処理やリサイズ戦略が必要
- プリンタ設定（余白あり/なし）は印刷時のシステム設定に依存する

## 11. 受け入れ基準（初期）
- ユーザーが複数画像を選択できる
- A4 / 2分割 / 4分割 / 6分割 / 8分割を切り替えてプレビュー反映される
- A4サイズPDFを生成し、iPhone上で保存または共有できる
- 画像が不自然に引き伸ばされない
- エラー時にユーザーが次の行動を判断できるメッセージが表示される

## 12. 今後の検討事項

- 余白サイズのカスタマイズ
- 複数ページPDF対応
- PWA化（オフライン性向上）

## 14. API不要の実装仕様（技術選定含む）
### 14.1 方針
- 完全フロントエンド実装（バックエンド/APIなし）
- 画像データは端末内で処理し、外部送信しない
- 高解像度維持のため、PDF埋め込み時は元画像バイトを優先して利用する

### 14.2 技術選定
- フレームワーク: `React + TypeScript`（`Vite`構成）
- UI: `Vanilla CSS`（軽量・依存最小）
- 状態管理: React標準（`useState` / `useReducer`）、必要時のみ `zustand`
- PDF生成: `pdf-lib`
- ファイル操作: Browser File API（`<input type="file" multiple accept="image/*">`）
- テスト: `Vitest`（単体） + `Playwright`（E2E最小）

選定理由:
- `pdf-lib` はJPEG/PNGをPDFに直接埋め込めるため、Canvas再圧縮を避けやすい
- API不要でiOS Chrome上で完結し、配布も静的ホスティングで可能

### 14.3 想定アーキテクチャ
```text
UI (React)
  -> 画像選択/設定 state
  -> レイアウト計算モジュール
  -> プレビューモジュール
  -> PDF生成モジュール (pdf-lib)
  -> 保存/共有 (Blob URL + Web Share API)
```

### 14.4 データモデル（TypeScript案）
```ts
type LayoutType = "a4_1" | "a4_2" | "a4_4" | "a4_6" | "a4_8";
type FitMode = "contain" | "cover";

interface SelectedImage {
  id: string;
  file: File;
  objectUrl: string;
  widthPx: number;
  heightPx: number;
  mimeType: "image/jpeg" | "image/png";
}

interface LayoutConfig {
  layoutType: LayoutType;
  fitMode: FitMode;
  marginMm: number; // default 5
  gapMm: number;    // default 2
}

interface SlotRectMm {
  xMm: number;
  yMm: number;
  widthMm: number;
  heightMm: number;
}
```

### 14.5 レイアウト仕様
- 用紙サイズ: A4固定（`210 x 297 mm`、縦向き）
- 分割グリッド:
  - 1分割: `1 x 1`
  - 2分割: `1 x 2`
  - 4分割: `2 x 2`
  - 6分割: `2 x 3`
  - 8分割: `2 x 4`
- 枠数を超える画像:
  - 同一レイアウトで自動的に次ページへ割り当てる
- 枠数未満の画像:
  - 空き枠は空白で出力する

レイアウト計算式:
- `usableWidthMm = 210 - marginMm * 2 - gapMm * (cols - 1)`
- `usableHeightMm = 297 - marginMm * 2 - gapMm * (rows - 1)`
- `slotWidthMm = usableWidthMm / cols`
- `slotHeightMm = usableHeightMm / rows`

### 14.6 画像配置・品質仕様
- 歪み防止: 常にアスペクト比維持
- `contain`:
  - 枠内に全体表示（余白あり）
- `cover`:
  - 枠を埋める（はみ出し部分はクリップ）
- 低解像度警告:
  - 各枠の物理サイズから300dpi相当の必要ピクセル数を算出
  - `画像px < 必要px` の場合、プレビューで警告表示

必要ピクセル計算:
- `requiredPx = mm / 25.4 * 300`

### 14.7 PDF生成仕様
- 単位変換:
  - `pt = mm * 72 / 25.4`
- `pdf-lib` でA4ページを作成し、画像を各スロットに描画
- 画像形式ごとの埋め込み:
  - JPEG: `embedJpg`
  - PNG: `embedPng`
- 出力:
  - `Blob`化し、保存または共有に渡す
- ファイル名:
  - `print-layout-YYYYMMDD-HHmm.pdf`

### 14.8 保存・共有仕様
- 第一手段: `navigator.share`（利用可能時）
- 代替手段: Blob URLダウンロードリンクを表示
- iOS挙動差分を考慮し、「共有」「保存」両導線を常に表示する

### 14.9 エラー処理仕様
- ファイル読込失敗: 該当画像を除外して通知
- 非対応形式: 選択時点で弾く
- PDF生成失敗: エラーメッセージと再試行ボタンを表示
- メモリ不足対策:
  - 1ページ単位で逐次処理して中間バッファを解放
