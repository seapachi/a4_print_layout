# EXPLANATION.md

## 概要
このプロジェクトは、端末内画像をA4印刷用PDFへ配置して出力するPoCです。
`index.html` 単体で動作し、現在は Hansi（和紙トーン）デザインの4画面フローを採用しています。

## 現在の実装範囲
- 画像選択: JPEG / PNG 複数選択
- 分割レイアウト: 1 / 2 / 4 / 6 / 8 分割
- 配置モード: `contain`（全体表示） / `cover`（トリミング）
- `Screen A` プレビュー: 選択画像の全件サムネイル表示（高さ可変）
- `Screen A` 表示簡素化: メトリクス/待機中表示を廃止し、警告/エラーは `errorBox` に集約
- `Screen C` プレビュー: A4比率に寄せた実画像プレビュー（ページ送り対応）
- PDF生成: `pdf-lib` を利用
- 出力: 共有（Web Share API）または保存（Blob URL）
- `Screen D` 表示: `PDFを開いて確認` ボタンからブラウザ標準PDFビューアを開く
- UI: Hansi固定、1カラム（右パネルなし）、ログは折りたたみ表示
- ヘッダー補助文: `スマホの写真をA4印刷PDFにかんたん配置（1/2/4/6/8分割）`
- 表示構成: スマホ外枠/上部ABCDタブ/端末ステータスバーを廃止し、内部レイアウト面のみをカード表示
- PCワイド補正: ヘッダー幅を `app-surface`（420px）に合わせて中央配置し、左ズレを抑制
- 固定CTA: `Screen A/B/C/D` すべてで下部CTAを固定し、本文は `sc-body` のみスクロール
- safe-area対応: 下部CTAと本文末尾に `env(safe-area-inset-bottom)` を反映し、iPhoneのホームインジケータ干渉を回避
- モバイル高さ対策: `100dvh` を基準にし、iOS Safariのアドレスバー伸縮でレイアウトが崩れにくい構成へ変更
- iPhone下余白対策: モバイル時の `app-surface` 高さを `clamp(560px, calc(100dvh - 140px), 620px)` へ見直し、A/B/CのCTAが画面下へ食い込みにくいように調整
- `Screen D` 初期簡素化: アプリ内埋め込みviewerを廃止し、必要時だけ標準PDFビューアへ遷移する構成に変更
- `Screen D` 完了表示: 完了アイコンは補助文の下へ独立配置し、中央寄せの完了感を復元
- `Screen B` 高さ最適化: 分割レイアウト候補を2列のコンパクトカードへ再構成し、iPhone縦画面でも配置モードとCTAまで初期表示で見渡せるように調整
- `Screen B` CTA高さ統一: `プレビューへ` / `戻る` は他画面と同じ共通ボタン高さへ戻し、専用の高さ上書きは廃止

## 画面構成
- `Screen A`: 画像選択、サムネイル、選択済み枚数、警告表示
- `Screen B`: 分割レイアウトと配置モードの選択
- `Screen B` 初期値: レイアウトは `4分割 (2x2)`、配置モードは `contain`（全体表示）
- `Screen C`: レイアウト反映済みの実画像プレビューとページ送り、PDF生成（配置ラベルは日本語表示）
- `Screen D`: 生成結果（ファイル名/ページ数）・PDF確認ボタン・保存/共有
- 画面遷移: `data-go` の次へ/戻るボタンで切替（戻る導線はB/C/Dの下部CTAに統一）
- 進行表示: 各画面上部のステップドットのみを利用

## 主要仕様
- 用紙サイズ: A4 `210 x 297 mm`
- 余白: `5 mm`
- ギャップ: `2 mm`
- レイアウト定義:
  - `1`: `1 x 1`
  - `2`: `1 x 2`
  - `4`: `2 x 2`
  - `6`: `2 x 3`
  - `8`: `2 x 4`

## 実装ポイント
### 1) 分割ロジックの一般化
`computeSlotsA4_8` を `computeSlots(layoutKey, marginMm, gapMm)` に置き換え、
選択された分割定義（列数/行数）からスロットを計算します。

### 2) 配置モードの追加
`buildPdfA4(images, slots, fitMode)` を導入しています。
- `contain`: 既存どおり縦横比維持で枠内に収める
- `cover`: 枠を埋めるスケールで描画し、`pdf-lib` のクリップで枠外を隠す

`cover` でクリップ演算子が使えない環境では安全側として `contain` 相当にフォールバックします。

### 3) プレビューとページ計算
1ページあたりのスロット数を選択レイアウトから動的に求め、
- プレビューのページ送り (`prev-pg` / `next-pg`)
を連動させています。

`Screen C` のプレビューはDOMベースで、`object-fit` により `contain/cover` を見た目に反映します。
表示ラベルは `全体表示` / `トリミング` に統一しています。

### 4) `Screen A` の全件サムネイル
- 6件固定プレースホルダを廃止し、選択した全画像を表示します。
- `ImageAsset` に `previewUrl` を保持し、再選択/リスタート時に `URL.revokeObjectURL` で解放します。
- 未選択時は空状態メッセージを全幅中央で表示します。

### 5) レイアウト設定ミニ図の見た目
`Screen B` のレイアウトミニ図（`.mg`）は縦長比率で表示し、A4縦方向の印象に寄せています。
分割定義（1/2/4/6/8）は従来どおりで、見た目のみ変更しています。

### 5-2) `Screen B` の2列カード化
- 分割候補は縦積みリストから、`2列 x 複数行` のコンパクトカードへ変更しました。
- 各カードは `ミニ図 / ラベル / 選択ラジオ` を横並びに整理し、1件あたりの高さを圧縮しています。
- `配置モード` もモバイルで2列維持とし、`Screen B` 全体が1画面に収まりやすいようにしています。
- CTA予約領域（`--sc-cta-space`）は `Screen B` 専用に再調整し、固定CTAを維持しつつ本文スクロールが出にくいバランスへ寄せました。
- ボタン高さは最終的に共通の `.btn` 定義へ揃え、Screen B だけを小さくする上書きは残していません。

### 6) PDF生成後のviewer表示
- 生成完了時に `Screen D` でBlob URLを確認リンクへ設定し、ブラウザ標準のPDFビューアへ開く構成にしています。
- iPhone Safari では `iframe + blob URL` の埋め込みPDF表示が不安定で、ズームやスクロールが崩れるため、アプリ内プレビューは採用していません。
- 実体PDF自体は正常なので、表示経路だけを安定する標準ビューアへ切り替えています。
- CTAは `共有する` と `保存する` を残し、PDF確認は本文側の補助操作として分離しています。
- 完了チェックは補助文直下の中央配置を維持し、完了状態を一目で認識しやすくしています。

### 7) 画面下部固定CTA
- 各画面コンテナ（`.sc`）を縦方向フレックスレイアウトで運用し、CTA（`.sc-cta`）を画面下部固定レイヤとして扱っています。
- 本文は `.sc-body` 側だけをスクロールさせるため、内容が増えても主要操作ボタンは常に同じ位置に表示されます。

### 8) 固定CTA + 本文スクロール分離
- 各画面を `.sc-body`（スクロール本文）と `.sc-cta`（画面下部固定）に分離しました。
- `Screen D` はCTAを `主操作`（共有/保存）と `補助操作`（戻る/もう一度作る）に分割し、PDF確認は本文側から標準PDFビューアへ遷移させています。
- 本文末尾は `--sc-cta-space` 分の余白を確保し、固定CTAに隠れないようにしています。

### 9) iPhone向け高さの再調整
- 旧構成ではモバイル時に `app-surface` の最小高さが強く、390px台のiPhoneでカード下端がビューポート外へ出ることがありました。
- そのためモバイル時はカード高さを可変にし、下部CTAの見切れよりも「カード全体が画面内へ収まること」を優先しています。
- `Screen D` は初期状態では結果要約と4操作を優先表示し、PDF確認は必要時のみ標準PDFビューアへ切り出しています。
- Safari系ブラウザでは埋め込みPDFよりも標準ビューアの方がスクロールと横幅フィットが安定するため、表示の信頼性を優先しています。

## JPEG縦横比崩れ対策（継続）
iPhone写真のEXIF Orientation差異に対して、次を継続実装しています。
1. JPEGはCanvas経由で正規化
2. EXIF Orientation（1-8）を解釈して回転・反転を適用
3. `createImageBitmap(..., { imageOrientation: "none" })` を優先
4. PDF描画時は埋め込み後の寸法を基準にスケール計算
